Basic Code Setup

<!--Head tag CSS-->
<link href="../videojs/skins/nuevo/videojs.min.css" rel="stylesheet" type="text/css" />

<!--Load videojs and nuevo plugin-->
<script src="../videojs/video.min.js"></script>
<script src="../videojs/nuevo.min.js"></script>

<!--Video tag setup-->
<video id="player_one" class="video-js vjs-fluid" controls preload playsinline poster="images/poster.jpg">
	<source src="path-to-live-stream.m3u8" type="application/x-mpegURL">
</video>

<!--Initialize the player and plugin-->
<script>
	var player = videojs('player_one)';
	player.nuevo({ 
		//option_name: option_value, option_name: value
	}); 
</script>
Nuevo plugin by default reconnects live stream when player paused and resumed to play. This was set due to many complains by videojs users about live stream stucked once it was paused. If you prefer to keep default videojs action ( does not reconnect stream when video resumed) you can set option liveReoload to false. 
<script>
	var player = videojs('player_one)';
	player.nuevo({ 
		liveReload:false
	}); 
</script>
You may also choose the option not to reconnect live stream when resumed after pause for certain amount of time (seconds) 
<script>
	var player = videojs('player_one)';
	player.nuevo({ 
		live Timeout:5
	}); 
</script>
This will reconnect live stream only when it was paused for longer than 5 seconds.
